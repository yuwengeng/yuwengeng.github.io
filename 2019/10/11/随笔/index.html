pipenv

创建虚拟环境

pipenv install

激活虚拟环境

pipenv shell

对一个新项目来说，你不必手动使用pipenv install来创建虚拟环境。当使用pipenv install xxx直接安装依赖包时，如果当前目录不包含虚拟环境，Pipenv会自动创建一个

在部署环境安装依赖

(venv)$ pip install -r requirements.txt # 安装requirement.txt中记录的依赖
使用Pipenv则只需要执行pipenv install，它会自动安装Pipfile中记录的依赖：

开发依赖

pip freeze > requirements.txt
$ pipenv install pytest --devPipenv会自动帮你管理依赖
在新的开发环境安装依赖时，也只需要在pipenv install命令后添加–dev选项即可一并安装开发依赖：

Pipenv甚至提供了读取和生成requirements.txt文件的功能（在使用pipenv install命令时它会自动读取requirements.txt文件安装依赖并生成Pipfile文件）
